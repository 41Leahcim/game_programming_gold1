Implementing a State Machine in Unity

Submission folder Gold 1 | Deadline: March 19, 23:59

Submission folder Gold 1 retake | Deadline: April 16, 23:59

Implement, using Unity, a State Machine without using existing solutions for State Machines (the animation state machine Unity provides, for example). In your Unity game, the goal is that the player should be able to control an object and change its state. 

Your game must contain a 3D environment/level containing several pieces of interesting geometry. Objects like ramps, arches, stairs, pillars are all valid. Using the ProBuilder package shown during the lectures is allowed and advised.

The player can move around the player GameObject in this level by using WASD or the arrow-keys. This Gameobject can just be a default Capsule Unity provides, or something more custom for additional points. The movement is managed by a State Machine. The State Machine contains at least 4 different movement states, for example: idle, moving, running and jumping. It is required that at least one of these states relies on another state and can only be entered when that state is active. For example: you can only enter the jump-state while you’re in the running-state. 

Make sure that the movement of the player Gameobject is frame rate-independent. It is also required to keep the player Gameobject in-frame while it’s moving around by implementing a follow-camera. Using a package like CineMachine to achieve this is allowed. 

The current active state should always be obvious to the player. Either by displaying the state name as an UI-element, or changing/updating some other visual element (for example, the colour of the player object). The submission is not valid if the changes in state can not be detected in this way.

Naive implementations (using a giant if-else statement, for example) of a State Machine are not sufficient for this assignment.

To submit:

  • The Unity project, including all code & assets, excluding the library folder!
  • A working build.

If you hand in a compressed file or files, the .zip extension is required. (.rar of anything else is not allowed)


Please put your name AND studentnumber in the file you submit.

Good luck!
